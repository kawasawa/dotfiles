# 翻訳併記

渡された対象（`$ARGUMENTS`）に翻訳を併記。原文が日本語なら英訳、それ以外なら日本語訳を付与。翻訳併記以外の作業は**不要**。

## 対象の判定（`$ARGUMENTS`）

- 既存のファイルパス → そのファイルに翻訳を直接追記。
- 翻訳対象の文そのもの → その文を翻訳し、応答に原文＋翻訳を併記して返す（ファイルは変更しない）。
- 空 → IDE で開いているファイルや直近で指定されたファイルを候補に `AskUserQuestion` ツールで確認。候補が無ければ対象（パスまたは文）の指定を依頼。
- パスらしき文字列がファイルとして存在しない場合は、文として扱う前に確認。

## 翻訳ルール

- 段落の境界は空行、または行頭の字下げ（全角空白など）で判定。
- 段落単位で言語を判定。日本語の段落には英訳、それ以外には日本語訳を併記。
- 翻訳は対応する原文ブロックの直後に配置。原文の各文と同じ行構成で対応。
- 翻訳ブロックの行頭は原文の字下げに合わせる。
- 翻訳ブロックと次の原文ブロックの間は空行 1 つで区切り。
- フォーマル寄りの自然な表現で訳出。原文のニュアンスとトーンを保持。
- 見出しや箇条書きの項目も同様。マーカー記号を保持したまま原文の直後へ併記。

## 非翻訳対象

- URL は変更禁止（リンクの表示テキストは翻訳、URL はそのまま保持）。
- コードは翻訳対象外。
  - フェンス・インラインともコードはそのまま保持。
  - 文中のインラインコードは、コードを保持したまま周囲の文を翻訳。

## 原文の扱い

- 原文の文言は**変更禁止**。誤字・脱字・文法不備も修正せずそのまま（人間が書くため不備が混在）。
- 原文の文字コードや改行は正規化禁止（メッセージ・ドキュメントツールからのコピーで不規則に混在）。
- 区切りの空行挿入以外、原文のレイアウトは改変禁止。

## 再実行時の冪等性（ファイル対象時）

- 翻訳併記済みの箇所は再併記せず、重複させない。
- 原文と既存翻訳に乖離がある場合のみ、その翻訳を更新。
- 併記済みの翻訳行を原文と誤認した**逆翻訳は禁止**。

## 動作例

### ファイル対象（原文）

````text
　日本語の段落1
日本語の文章1-1
日本語の文章1-2
　日本語の段落2
日本語の文書2-1

[ソースコード](https://example.com)

`console.log("こんにちは、世界")`

```ts
  const message = "こんにちは、世界";
```

　日本語の段落3
日本語の文書3-1
````

### ファイル対象（翻訳併記後）

````text
　日本語の段落1
日本語の文章1-1
日本語の文章1-2
  English translation of paragraph 1
English translation of sentence 1-1
English translation of sentence 1-2

　日本語の段落2
日本語の文書2-1
  English translation of paragraph 2
English translation of sentence 2-1

[Source code](https://example.com)

`console.log("こんにちは、世界")`

```ts
  const message = "こんにちは、世界";
```

　日本語の段落3
日本語の文書3-1
  English translation of paragraph 3
English translation of sentence 3-1
````

### ファイル対象（見出し・箇条書き）

原文:

```text
## 見出し
- 箇条書き項目
```

併記後:

```text
## 見出し
## Heading

- 箇条書き項目
- List item
```

### テキスト対象（応答に併記）

入力: `今日はいい天気ですね。とても気持ちのいい朝です。`

応答:

```text
今日はいい天気ですね。とても気持ちのいい朝です。
It is a beautiful day today. What a pleasant morning.
```
